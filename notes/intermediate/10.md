# File Uploads & Xlsxir: Processing Excel Files in LiveView

## The Big Picture

```
User uploads .xlsx file
        │
        ▼
┌─────────────────────────────────┐
│  LiveView receives file         │
│  Stores in temp location        │
└─────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────┐
│  consume_uploaded_entries()     │
│  Calls YOUR callback function   │
└─────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────┐
│  Xlsxir.multi_extract()         │
│  Unzips .xlsx, parses XML       │

└─────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────┐
│  Xlsxir.get_list()              │
│  Returns all rows as list       │
└─────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────┐
│  Xlsxir.close()                 │
│  Cleanup! Delete ETS table      │
└─────────────────────────────────┘
```

## Part 1: Setting Up File Upload in LiveView

### Step 1: Allow uploads in mount

```elixir
def mount(_params, _session, socket) do
  {:ok,
   socket
   |> assign(:rows, [])
   |> allow_upload(:spreadsheet,      # Name of the upload
       accept: ~w(.xlsx),             # Only accept .xlsx files
       max_entries: 1,                # Max 1 file at a time
       max_file_size: 10_000_000      # 10MB limit
     )}
end
```

```heex

</form>
```

  # Process the uploaded file

## Part 2: consume_uploaded_entries Explained

### What is it?

`consume_uploaded_entries` loops through uploaded files and calls YOUR callback function for each one.

### JS Equivalent

```javascript
// JavaScript
const results = files.map(file => processFile(file));

// Elixir
results = consume_uploaded_entries(socket, :spreadsheet, &process_file/2)
```

### The Three Arguments

```elixir
consume_uploaded_entries(socket, :spreadsheet, &parse_excel_file/2)
#                        │       │              │
#                        │       │              └── Your callback function
#                        │       └── Upload name (from allow_upload)
#                        └── The LiveView socket
```

### What does `&parse_excel_file/2` mean?

```elixir
&parse_excel_file/2
# │              │
# │              └── Takes 2 arguments
# └── Reference to function (not calling it)

# JavaScript equivalent:
// parseExcelFile (without parentheses - just the reference)
```

### The Callback Function

```elixir
def parse_excel_file(%{path: file_path}, entry) do
  #                   │                   │
  #                   │                   └── UploadEntry struct (metadata)
  #                   └── Map with temp file path

  # entry.client_name → "products.xlsx" (original filename)
  # entry.client_size → 8623 (bytes)
  # file_path → "/tmp/plug-1234/live_view_upload-5678"

  {:ok, result}  # Must return {:ok, ...} or {:error, ...}
end
```

### Complete Example

```elixir
def handle_event("import", _params, socket) do
  # Returns a LIST of results (one per file)
  uploaded_results = consume_uploaded_entries(
    socket,
    :spreadsheet,
    &parse_excel_file/2
  )

  # If you uploaded 1 file, uploaded_results = [result]
  # If you uploaded 3 files, uploaded_results = [result1, result2, result3]

  case uploaded_results do
    [] ->
      {:noreply, put_flash(socket, :error, "No file uploaded")}
    [rows] ->
      {:noreply, assign(socket, :rows, rows)}
  end
end

def parse_excel_file(%{path: file_path}, _entry) do
  # Your processing logic here
  {:ok, [["row1"], ["row2"]]}  # Whatever you return here ends up in the list
end
```

## Part 3: Xlsxir - Reading Excel Files

### What is Xlsxir?

Excel files (.xlsx) are actually **ZIP files containing XML**. Xlsxir:
1. Unzips the file
2. Parses the XML
3. Stores data in ETS (in-memory database)

### The Three Functions

| Function | Purpose | Returns |
|----------|---------|---------|
| `multi_extract(path, sheet)` | Parse file, store in ETS | `{:ok, table_id}` |
| `get_list(table_id)` | Get all rows | `[[row1], [row2], ...]` |
| `close(table_id)` | Delete ETS table | `:ok` |

### Basic Usage

```elixir
# Step 1: Parse the file (stores in ETS, returns table_id)
{:ok, table_id} = Xlsxir.multi_extract(file_path, 0)
#                                                  └── Sheet index (0 = first sheet)

# Step 2: Get the data
all_rows = Xlsxir.get_list(table_id)
# Returns: [["Name", "Price"], ["Widget", 10.99], ["Gadget", 24.99]]

# Step 3: CLEANUP! Delete the ETS table
Xlsxir.close(table_id)
```

## Part 4: What is ETS?

**ETS = Erlang Term Storage**

Think of it as a super-fast in-memory database built into Erlang/Elixir.

```
┌─────────────────────────────────────────────────────────┐
│                     YOUR ELIXIR APP                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────────────────────────────────────────┐   │
│   │              ETS TABLE (in RAM)                 │   │
│   │  ┌─────────────────────────────────────────┐    │   │
│   │  │ Row 0: ["Name", "Price", "Qty"]         │    │   │
│   │  │ Row 1: ["Widget", 10.99, 5]             │    │   │
│   │  │ Row 2: ["Gadget", 24.99, 3]             │    │   │
│   │  └─────────────────────────────────────────┘    │   │
│   │                                                 │   │
│   │  table_id = reference to access this table      │   │
│   └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| Feature | Description |
|---------|-------------|
| Speed | Extremely fast (in-memory) |
| Storage | Lives in RAM, not on disk |
| Access | Any process can read/write with table_id |
| Lifecycle | **You must manually delete it!** |

### Why Use ETS?

For large Excel files, ETS lets you:
- Stream rows one at a time (not load everything into memory)
- Query specific rows: `Xlsxir.get_row(table_id, 5)`
- Query specific cells: `Xlsxir.get_cell(table_id, "A1")`

## Part 5: try/after for Guaranteed Cleanup

### The Problem

```elixir
{:ok, table_id} = Xlsxir.multi_extract(file_path, 0)
rows = Xlsxir.get_list(table_id)  # What if this crashes?
Xlsxir.close(table_id)            # This never runs! MEMORY LEAK!
```

### The Solution: try/after

```elixir
{:ok, table_id} = Xlsxir.multi_extract(file_path, 0)

try do
  rows = Xlsxir.get_list(table_id)
  {:ok, rows}
rescue
  error ->
    Logger.error("Failed: #{inspect(error)}")
    {:ok, {:error, :read_failed}}
after
  # ALWAYS runs, even if error occurs!
  Xlsxir.close(table_id)
end
```

### JS Equivalent

```javascript
// JavaScript try/finally
let tableId;
try {
  tableId = xlsxir.extract(filePath);
  const rows = xlsxir.getList(tableId);
  return rows;
} catch (error) {
  console.error('Failed:', error);
  return { error: 'read_failed' };
} finally {
  // ALWAYS runs
  xlsxir.close(tableId);
}
```

## Part 6: Complete Working Example

```elixir
defmodule HelloWeb.ProductsUploadLive do
  use HelloWeb, :live_view
  require Logger

  # ═══════════════════════════════════════════════════════════════
  # MOUNT - Setup the upload
  # ═══════════════════════════════════════════════════════════════
  def mount(_params, _session, socket) do
    {:ok,
     socket
     |> assign(:headers, [])
     |> assign(:rows, [])
     |> allow_upload(:spreadsheet,
         accept: ~w(.xlsx),
         max_entries: 1,
         max_file_size: 10_000_000
       )}
  end

  # ═══════════════════════════════════════════════════════════════
  # HANDLE EVENTS
  # ═══════════════════════════════════════════════════════════════
  def handle_event("validate", _params, socket) do
    {:noreply, socket}
  end

  def handle_event("import", _params, socket) do
    # consume_uploaded_entries calls parse_excel_file for each file
    uploaded_results = consume_uploaded_entries(
      socket,
      :spreadsheet,
      &parse_excel_file/2
    )

    case uploaded_results do
      [] ->
        {:noreply, put_flash(socket, :error, "No file uploaded")}

      [{:error, reason}] ->
        {:noreply, put_flash(socket, :error, "Failed: #{reason}")}

      [[headers | data_rows]] ->
        {:noreply,
         socket
         |> assign(:headers, headers)
         |> assign(:rows, data_rows)
         |> put_flash(:info, "Imported #{length(data_rows)} rows")}
    end
  end

  # ═══════════════════════════════════════════════════════════════
  # PARSE EXCEL FILE - Called by consume_uploaded_entries
  # ═══════════════════════════════════════════════════════════════
  defp parse_excel_file(%{path: file_path}, entry) do
    case Xlsxir.multi_extract(file_path, 0) do
      {:ok, table_id} ->
        try do
          rows = Xlsxir.get_list(table_id)
          {:ok, rows}
        rescue
          error ->
            Logger.error("Read failed: #{inspect(error)}")
            {:ok, {:error, :read_failed}}
        after
          Xlsxir.close(table_id)  # ALWAYS cleanup!
        end

      {:error, reason} ->
        Logger.error("Open failed: #{inspect(reason)}")
        {:ok, {:error, :open_failed}}
    end
  end

  # ═══════════════════════════════════════════════════════════════
  # RENDER
  # ═══════════════════════════════════════════════════════════════
  def render(assigns) do
    ~H"""
    <h1>Products Upload</h1>

    <form phx-change="validate" phx-submit="import">
      <.live_file_input upload={@uploads.spreadsheet} />
      <button type="submit">Import</button>
    </form>

    <table :if={@rows != []}>
      <thead>
        <tr>
          <th :for={header <- @headers}>{header}</th>
        </tr>
      </thead>
      <tbody>
        <tr :for={row <- @rows}>
          <td :for={cell <- row}>{cell}</td>
        </tr>
      </tbody>
    </table>
    """
  end
end
```

## Part 7: Library Analogy

Think of Xlsxir like a library:

| Action | Library Analogy |
|--------|-----------------|
| `multi_extract()` | Check out a book, get a library card number |
| `table_id` | Your library card number |
| `get_list()` | Use your card to read the book |
| `close()` | Return the book |

**If you never return the book (forget `close()`), the library fills up (memory leak)!**

## Quick Reference

```elixir
# 1. Setup upload in mount
|> allow_upload(:name, accept: ~w(.xlsx), max_entries: 1)

# 2. Process uploads
consume_uploaded_entries(socket, :name, &callback/2)

# 3. Callback receives
def callback(%{path: file_path}, entry) do
  # file_path = temp file location
  # entry.client_name = original filename
  {:ok, result}
end

# 4. Xlsxir workflow
{:ok, table_id} = Xlsxir.multi_extract(path, 0)
rows = Xlsxir.get_list(table_id)
Xlsxir.close(table_id)  # Don't forget!
```

---

## TL;DR

1. **allow_upload** - Enable file uploads in mount
2. **consume_uploaded_entries** - Loop through files, call your callback
3. **Xlsxir.multi_extract** - Parse Excel, store in ETS, get table_id
4. **Xlsxir.get_list** - Read all rows from ETS
5. **Xlsxir.close** - DELETE the ETS table (prevent memory leak!)
6. **try/after** - Guarantee cleanup runs even on errors
