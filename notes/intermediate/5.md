# Complete Guide on Using Ecto To Add Tables

## The Generator Command

```bash
mix phx.gen.schema Context.Schema table_name field:type field:type ...
```

### Breaking it down:

| Part | Example | Meaning |
|------|---------|---------|
| `Context` | `Accounts` | The module namespace (folder) |
| `Schema` | `User` | The schema name |
| `table_name` | `users` | The actual PostgreSQL table name |
| `field:type` | `name:string` | Column name and type |

### Common field types:

| Type | PostgreSQL | Example |
|------|------------|---------|
| `:string` | `varchar` | `name:string` |
| `:text` | `text` | `bio:text` |
| `:integer` | `integer` | `age:integer` |
| `:decimal` | `decimal` | `price:decimal` |
| `:boolean` | `boolean` | `active:boolean` |
| `:datetime` | `timestamp` | `published_at:datetime` |
| `:references` | `foreign key` | `user_id:references:users` |

## Example: Creating a Users Table

### 1. Run the generator

```bash
mix phx.gen.schema Accounts.User users name:string email:string role:string
```

### 2. What gets created

**Schema file:** `lib/hello/accounts/user.ex`
```elixir
defmodule Hello.Accounts.User do
  use Ecto.Schema
  import Ecto.Changeset

  schema "users" do
    field :name, :string
    field :email, :string
    field :role, :string

    timestamps()
  end

  @doc false
  def changeset(user, attrs) do
    user
    |> cast(attrs, [:name, :email, :role])
    |> validate_required([:name, :email, :role])
  end
end
```

**Migration file:** `priv/repo/migrations/TIMESTAMP_create_users.exs`
```elixir
defmodule Hello.Repo.Migrations.CreateUsers do
  use Ecto.Migration

  def change do
    create table(:users) do
      add :name, :string
      add :email, :string
      add :role, :string

      timestamps()
    end
  end
end
```

### 3. Run the migration

```bash
mix ecto.migrate
```

## The Two Files Explained

### Schema (`lib/hello/accounts/user.ex`)

**Purpose:** Defines how Elixir talks to the database table.

```elixir
schema "users" do          # Maps to "users" table
  field :name, :string     # Maps to "name" column
  field :email, :string
  field :role, :string
  timestamps()             # Adds inserted_at, updated_at
end
```

### Migration (`priv/repo/migrations/...`)

**Purpose:** Actually creates/modifies the database table.

```elixir
def change do
  create table(:users) do   # CREATE TABLE users
    add :name, :string      # name VARCHAR
    add :email, :string
    add :role, :string
    timestamps()            # inserted_at, updated_at TIMESTAMP
  end
end
```

## Workflow Summary

```
1. Generate
   mix phx.gen.schema Accounts.User users name:string email:string role:string

2. (Optional) Edit migration to add constraints
   # Add unique index, not null, etc.

3. Run migration
   mix ecto.migrate

4. Create a Context module to interact with the schema
   # lib/hello/accounts.ex
```

## Adding a Context (Optional but Recommended)

The generator creates the schema, but you often want a **context** module for business logic:

**`lib/hello/accounts.ex`**
```elixir
defmodule Hello.Accounts do
  alias Hello.Repo
  alias Hello.Accounts.User

  def list_users do
    Repo.all(User)
  end

  def get_user!(id) do
    Repo.get!(User, id)
  end

  def create_user(attrs \\ %{}) do
    %User{}
    |> User.changeset(attrs)
    |> Repo.insert()
  end

  def update_user(%User{} = user, attrs) do
    user
    |> User.changeset(attrs)
    |> Repo.update()
  end

  def delete_user(%User{} = user) do
    Repo.delete(user)
  end
end
```

## Quick Reference

| Task | Command |
|------|---------|
| Generate schema + migration | `mix phx.gen.schema Context.Schema table fields...` |
| Run pending migrations | `mix ecto.migrate` |
| Rollback last migration | `mix ecto.rollback` |
| Create database | `mix ecto.create` |
| Drop database | `mix ecto.drop` |
| Reset (drop + create + migrate + seed) | `mix ecto.reset` |
