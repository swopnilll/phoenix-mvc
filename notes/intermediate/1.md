# Phoenix LiveView for React Developers

## Mental Model: The Big Shift

| React (SPA) | Phoenix LiveView |
|---|---|
| Client renders everything | Server renders everything |
| State lives in browser (`useState`, Redux) | State lives on server (`assigns`) |
| REST API calls to get data | Data fetched directly in Elixir |
| JSON over HTTP | WebSocket connection (real-time) |
| JSX compiles to JS | HEEX compiles to HTML |
| Components are JS functions | Components are Elixir functions |

> **Key insight:** In LiveView, there's no "fetch data, update state, re-render" cycle. The server already has the data, renders HTML, and pushes only the changed parts over WebSocket.

## Anatomy of a LiveView

```elixir
defmodule MyAppWeb.ProductsLive do
  use MyAppWeb, :live_view           # 1. This is a LiveView

  def mount(_params, _session, socket) do   # 2. Like useEffect on mount
    {:ok, socket |> assign(:page_title, "Products")}
  end

  def render(assigns) do              # 3. Like your JSX return
    ~H"""
    <div>Your HTML here</div>
    """
  end
end
```

### React vs LiveView Lifecycle

| React | LiveView |
|---|---|
| `useState()` | `assign(socket, :key, value)` |
| `useEffect([], ...)` on mount | `mount/3` function |
| `return <JSX />` | `render/1` with `~H""" """` |
| `onClick={handler}` | `phx-click="event_name"` |
| Props: `<Comp data={x} />` | Assigns: `<.comp data={@x} />` |

## Template Syntax (HEEX)

HEEX = HTML + Elixir Expressions.

| JSX (React) | HEEX (LiveView) |
|---|---|
| `{variable}` | `{@variable}` (note the `@`) |
| `{items.map(i => ...)}` | `:for={i <- @items}` |
| `{condition && <div/>}` | `:if={condition}` |
| `<Component prop={x} />` | `<.component prop={@x} />` (note the `.`) |
| `className="..."` | `class="..."` |

## Tabs Example

### Tab Structure

```heex
<.tabs>
  <.tabs_list variant="segmented">
    <:tab class="flex-1" name="orders">
      <.icon name="hero-shopping-cart" /> Orders
    </:tab>
    <:tab class="flex-1" name="products">
      <.icon name="hero-cube" /> Products
    </:tab>
  </.tabs_list>

  <.tabs_panel active name="orders" class="mt-6">
    <!-- Orders tab content -->
  </.tabs_panel>

  <.tabs_panel name="products" class="mt-6">
    <!-- Products tab content -->
  </.tabs_panel>
</.tabs>
```

Key concepts:
- `<.tabs>` — a component (the `.` means it's a function component)
- `<:tab>` — a slot (like React's `children` but named)
- `name="..."` — identifies which tab this is
- Only one panel should have the `active` attribute initially

## Buttons: Navigation vs Actions

### Navigation button (like React Router's `<Link>`)

```heex
<.button variant="solid" title="New Product" navigate={~p"/products/new"}>
  New Product
</.button>
```

### Action button (triggers server-side logic)

```heex
<.button variant="solid" phx-click="import_sheet">
  Import Sheet
</.button>
```

Handle it on the server:

```elixir
def handle_event("import_sheet", _params, socket) do
  # Your logic here
  {:noreply, socket}
end
```

> `phx-click` is like `onClick` but sends an event to the server instead of running client-side JS.

## State Management Comparison

**React**

```javascript
const [products, setProducts] = useState([]);

useEffect(() => {
  fetch('/api/products')
    .then(r => r.json())
    .then(data => setProducts(data));
}, []);
```

**LiveView**

```elixir
def mount(_params, _session, socket) do
  {:ok, assign(socket, :products, Store.list_products())}
end
```

No fetch needed — you're on the server, so just call your database directly.

## Event Handling Comparison

**React**

```jsx
<button onClick={() => setCount(count + 1)}>
  Clicked {count} times
</button>
```

**LiveView**

```heex
<%# In template %>
<button phx-click="increment">
  Clicked {@count} times
</button>
```

```elixir
# In module
def handle_event("increment", _params, socket) do
  {:noreply, assign(socket, :count, socket.assigns.count + 1)}
end
```

## Routing

Routes are defined in `router.ex`. LiveView routes use `live/2` instead of `get/3`:

```elixir
scope "/", MyAppWeb do
  pipe_through :browser

  live "/", HomeLive
  live "/orders", OrdersLive
  live "/orders/:id", OrdersLive.Show
  live "/products", ProductsLive
  live "/products/new", ProductsLive.New
end
```

Each route maps to a LiveView module in `lib/my_app_web/live/`.

## Learning Path

1. **Read** a LiveView module top to bottom — they're typically short and very readable
2. **Add a tab** — add a `<:tab>` slot inside `<.tabs_list>`, then a matching `<.tabs_panel>`
3. **Add a button** — use `navigate={~p"/path"}` for links, `phx-click="event"` for actions
4. **Handle events** — add `def handle_event("event_name", _params, socket)` to your module
5. **Load data** — fetch in `mount/3` and assign to socket; no API layer needed
