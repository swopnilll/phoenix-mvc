# Phoenix Router — A Practical Guide for React Developers

---

## Part 1: Let's Build an Ordering System from Scratch

Imagine we're building a simple ordering system. You've just run `mix phx.new ordering_system` and your `router.ex` looks like this — basically empty:

```elixir
defmodule OrderingSystemWeb.Router do
  use OrderingSystemWeb, :router

  pipeline :browser do
    plug :accepts, ["html"]
    plug :fetch_session
    plug :fetch_live_flash
    plug :put_root_layout, html: {OrderingSystemWeb.Layouts, :root}
    plug :protect_from_forgery
    plug :put_secure_browser_headers
  end

  scope "/", OrderingSystemWeb do
    pipe_through :browser
    # nothing here yet
  end
end
```

Let's build it up step by step.

---

### Step 1 — A Home Page that says "Home"

**What we want:** Visit `/` and see an `<h1>` that says "Home".

#### 1a. Add the route

```elixir
scope "/", OrderingSystemWeb do
  pipe_through :browser

  live "/", HomeLive        # <-- add this
end
```

#### 1b. Create the LiveView file

Create `lib/ordering_system_web/live/home_live.ex`:

```elixir
defmodule OrderingSystemWeb.HomeLive do
  use OrderingSystemWeb, :live_view

  def render(assigns) do
    ~H"""
    <h1>Home</h1>
    """
  end
end
```

That's it. Visit `http://localhost:4000/` and you'll see **Home** in an `<h1>`.

> **React equivalent:** This is like a React component returned from a route:
> ```jsx
> // <Route path="/" element={<Home />} />
> function Home() { return <h1>Home</h1> }
> ```
> The key difference: in LiveView, this renders on the **server** and is delivered over a WebSocket. No client-side JS rendering.

---

### Step 2 — A Products Page with just a title

**What we want:** Visit `/products` and see `<h1>Products</h1>`.

#### 2a. Add the route

```elixir
scope "/", OrderingSystemWeb do
  pipe_through :browser

  live "/", HomeLive
  live "/products", ProductsLive        # <-- add this
end
```

#### 2b. Create the LiveView file

Create `lib/ordering_system_web/live/products_live.ex`:

```elixir
defmodule OrderingSystemWeb.ProductsLive do
  use OrderingSystemWeb, :live_view

  def render(assigns) do
    ~H"""
    <h1>Products</h1>
    """
  end
end
```

Visit `http://localhost:4000/products` — you'll see **Products**.

---

### Step 3 — Add a "Go to Products" button on the Home page

**What we want:** The Home page has a button that navigates to `/products`.

Update `home_live.ex`:

```elixir
defmodule OrderingSystemWeb.HomeLive do
  use OrderingSystemWeb, :live_view

  def render(assigns) do
    ~H"""
    <h1>Home</h1>
    <.link navigate={~p"/products"}>
      <button>Go to Products</button>
    </.link>
    """
  end
end
```

> **What is `<.link navigate>`?**
> This is Phoenix's built-in component for client-side navigation between LiveViews. It's like React Router's `<Link to="/products">`. The `~p"/products"` is a verified route — Phoenix will throw a compile error if the path doesn't exist in your router.
>
> Because both `/` and `/products` are in the **same** `live_session` (we'll explain this below), clicking the button reuses the existing WebSocket connection — no full page reload, just like React Router.

---

### Step 4 — The Full Router File (as it stands now)

```elixir
defmodule OrderingSystemWeb.Router do
  use OrderingSystemWeb, :router

  # ─── PIPELINES ────────────────────────────────────────────────────────────
  # Think of these as middleware chains, like Express middleware or
  # React Router loaders. Every request through a pipeline runs all the plugs.

  pipeline :browser do
    plug :accepts, ["html"]
    plug :fetch_session
    plug :fetch_live_flash
    plug :put_root_layout, html: {OrderingSystemWeb.Layouts, :root}
    plug :protect_from_forgery
    plug :put_secure_browser_headers
  end

  # ─── ROUTES ───────────────────────────────────────────────────────────────

  scope "/", OrderingSystemWeb do
    pipe_through :browser

    live "/", HomeLive
    live "/products", ProductsLive
  end
end
```

Now let's understand every concept in depth.

---


| Concept | What it is | React equivalent |
|---|---|---|
| `pipeline` | Middleware chain | Express `app.use()` / loader functions |
| `scope` | Route group with path prefix + namespace | `<Route path="/admin">` wrapper |
| `get` / `post` | Plain HTTP route | REST API endpoint |
| `live` | WebSocket route (LiveView) | React component on a route, but server-rendered |
| `live_session` | Shared WebSocket boundary + auth guards | Auth wrapper around a group of routes |
| Third `live` arg (`:index`, `:new`) | Action — controls modal/panel state in one LiveView | URL param driving modal open/closed |
| `~p"/products"` | Verified route path | `to="/products"` — but compile-checked |
| `<.link navigate>` | Client-side LiveView navigation | React Router `<Link>` |
