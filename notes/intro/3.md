# Phoenix Controller Naming Conventions

## The Magical Connection

When you call `render(conn, :home)` in a controller, Phoenix automatically finds the right template. How?

```
PageController           →    PageHTML            →    page_html/home.html.heex
      ↓                           ↓                              ↓
render(conn, :home)      embed_templates        The actual HTML template
                         "page_html/*"
```

---

## The Naming Convention

Phoenix follows a strict naming pattern:

| Controller | HTML Module | Template Folder |
|------------|-------------|-----------------|
| `PageController` | `PageHTML` | `page_html/` |
| `InvestorController` | `InvestorHTML` | `investor_html/` |
| `ProductController` | `ProductHTML` | `product_html/` |

**The rule:** Remove `Controller`, add `HTML`. The folder name is the snake_case version.

---

## How `render(conn, :home)` Works

### Step 1: Router calls Controller

```elixir
# lib/hello_web/router.ex
get "/home", PageController, :home
```

This calls `PageController.home/2`.

### Step 2: Controller calls render

```elixir
# lib/hello_web/controllers/page_controller.ex
defmodule HelloWeb.PageController do
  use HelloWeb, :controller

  def home(conn, _params) do
    render(conn, :home)   # ← What happens here?
  end
end
```

When you call `render(conn, :home)`:
1. Phoenix looks at the module name: `HelloWeb.PageController`
2. It replaces `Controller` with `HTML`: `HelloWeb.PageHTML`
3. It calls `PageHTML.home(assigns)` function

### Step 3: HTML Module provides the template

```elixir
# lib/hello_web/controllers/page_html.ex
defmodule HelloWeb.PageHTML do
  use HelloWeb, :html

  embed_templates "page_html/*"   # ← This is the magic line!
end
```

The `embed_templates "page_html/*"` macro:
- Scans the `page_html/` folder for `.heex` files
- Creates a function for each file:
  - `home.html.heex` → `def home(assigns)`
  - `about.html.heex` → `def about(assigns)`

### Step 4: Template renders HTML

```heex
<%# lib/hello_web/controllers/page_html/home.html.heex %>
<h1>Welcome Home!</h1>
```

## Complete Example: Investors Page

Let's build a page to list investors and view individual investor details.

### File Structure

```
lib/hello_web/
├── router.ex
└── controllers/
    ├── investor_controller.ex      # The controller
    ├── investor_html.ex            # The HTML module
    └── investor_html/              # Template folder
        ├── index.html.heex         # List all investors
        └── show.html.heex          # Show one investor
```

---

### Step 1: Define Routes

```elixir
# lib/hello_web/router.ex
scope "/", HelloWeb do
  pipe_through :browser

  get "/investors", InvestorController, :index        # List all
  get "/investors/:id", InvestorController, :show     # Show one
end
```

---

### Step 2: Create the Controller

```elixir
# lib/hello_web/controllers/investor_controller.ex
defmodule HelloWeb.InvestorController do
  use HelloWeb, :controller

  # Fake data for demo (normally from database)
  @investors [
    %{id: 1, name: "Warren Buffett", company: "Berkshire Hathaway", invested: 5_000_000},
    %{id: 2, name: "Cathie Wood", company: "ARK Invest", invested: 2_500_000},
    %{id: 3, name: "Peter Thiel", company: "Founders Fund", invested: 10_000_000}
  ]

  # GET /investors
  def index(conn, _params) do
    render(conn, :index, investors: @investors)
    #                    ↑
    #            Pass data to template as assigns
  end

  # GET /investors/:id
  def show(conn, %{"id" => id}) do
    investor = Enum.find(@investors, &(&1.id == String.to_integer(id)))

    case investor do
      nil ->
        conn
        |> put_flash(:error, "Investor not found")
        |> redirect(to: ~p"/investors")

      investor ->
        render(conn, :show, investor: investor)
    end
  end
end
```

**Key points:**
- `render(conn, :index, investors: @investors)` - Pass data as keyword list
- Data becomes available as `@investors` in the template
- `:index` atom tells Phoenix to use `index.html.heex`
- `:show` atom tells Phoenix to use `show.html.heex`

---

### Step 3: Create the HTML Module

```elixir
# lib/hello_web/controllers/investor_html.ex
defmodule HelloWeb.InvestorHTML do
  use HelloWeb, :html

  embed_templates "investor_html/*"
end
```

That's it! The `embed_templates` macro does all the work.

### Step 4: Create the Templates

**List all investors:**


## How Data Flows

```
Router                     Controller                    Template
──────                     ──────────                    ────────
GET /investors      →      index(conn, params)     →     @investors available
                           render(conn, :index,
                             investors: @investors)

GET /investors/2    →      show(conn, %{"id" => "2"})  → @investor available
                           render(conn, :show,
                             investor: investor)
```

---

## The Connection (`conn`)

The `conn` struct carries everything about the request and response:

```elixir
def show(conn, params) do
  # What's in conn?
  conn.params        # %{"id" => "2"}
  conn.path_info     # ["investors", "2"]
  conn.method        # "GET"
  conn.assigns       # %{} - your data goes here

  # What's in params? (second argument)
  params             # %{"id" => "2"} - same as conn.params
end
```

When you call `render(conn, :show, investor: investor)`:
1. `investor: investor` is merged into `conn.assigns`
2. Template receives `assigns` map
3. `@investor` in template reads from `assigns.investor`

## Alternative: Inline Templates

Instead of separate `.heex` files, you can define templates inline:

```elixir
# lib/hello_web/controllers/investor_html.ex
defmodule HelloWeb.InvestorHTML do
  use HelloWeb, :html

  # Load templates from files
  embed_templates "investor_html/*"

  # OR define inline for simple pages
  def simple(assigns) do
    ~H"""
    <h1>Simple inline template</h1>
    <p>Investor: {@name}</p>
    """
  end
end
```

Use inline for very simple templates. Use files for anything complex.

---

## Summary

| Concept | What It Does |
|---------|--------------|
| `render(conn, :action)` | Calls `{Module}HTML.action(assigns)` |
| `embed_templates "folder/*"` | Creates functions from `.heex` files |
| `@variable` in template | Reads from `assigns.variable` |
| Third argument to render | Merges into `conn.assigns` |

**The naming convention:**
```
{Name}Controller  →  {Name}HTML  →  {name}_html/{action}.html.heex
```

This convention is why Phoenix "just works" - no configuration needed, just follow the pattern!
