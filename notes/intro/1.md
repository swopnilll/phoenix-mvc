# Phoenix/Elixir for Express/Node.js Developers

## The Big Picture

| Express/Node | Phoenix/Elixir | Your File |
|-------------|----------------|-----------|
| `package.json` | `mix.exs` | `/mix.exs` |
| `node_modules/` | `deps/` | (created after `mix deps.get`) |
| `app.js` or `server.js` | `application.ex` | `/lib/hello/application.ex` |
| Express `app.use()` middleware | Plug pipeline in Endpoint | `/lib/hello_web/endpoint.ex` |
| Express Router | Phoenix Router | `/lib/hello_web/router.ex` |
| Controllers | Controllers | `/lib/hello_web/controllers/` |
| `.env` files | `config/*.exs` | `/config/` folder |

---

## 1. `mix.exs` = Your `package.json`

This defines:
- App name and version
- Dependencies (like `phoenix`, `ecto`, `postgrex`)
- Scripts (called "aliases" in Elixir)

**Key aliases you'll use:**
```bash
mix setup          # Like "npm install && npm run db:setup"
mix phx.server     # Like "npm start" or "node server.js"
mix ecto.create    # Creates the database
mix ecto.migrate   # Runs migrations
```

---

## 2. `lib/hello/application.ex` = Your `server.js` Entry Point

In Express you might have:
```javascript
const app = express();
app.listen(3000);
```

In Phoenix, `application.ex` starts a **supervision tree** (services that auto-restart if they crash):
```
1. Telemetry (metrics)
2. Repo (database connection pool)  ← Like your Sequelize/Knex connection
3. PubSub (real-time messaging)     ← Like Socket.io
4. Endpoint (HTTP server)           ← Like Express app.listen()
```

---

## 3. `lib/hello_web/endpoint.ex` = Express Middleware Chain

In Express:
```javascript
app.use(express.json());
app.use(express.static('public'));
app.use(session({...}));
app.use(router);
```

In Phoenix, the Endpoint does the same thing with "Plugs":
```elixir
plug Plug.Static          # Serve static files
plug Plug.Parsers         # Parse JSON, form data
plug Plug.Session         # Session handling
plug HelloWeb.Router      # Your routes
```

---

## 4. `lib/hello_web/router.ex` = Your Express Router

**Express:**
```javascript
router.get('/', (req, res) => res.render('home'));
router.get('/api/users', userController.index);
```

**Phoenix:**
```elixir
get "/", PageController, :home
# API routes would go here
```

The router also defines **pipelines** (middleware groups):
- `:browser` - For HTML pages (session, CSRF, etc.)
- `:api` - For JSON APIs (no session needed)

---

## 5. Controllers = Same Concept!

**Express:**
```javascript
exports.home = (req, res) => {
  res.render('home', { title: 'Welcome' });
};
```

**Phoenix (`page_controller.ex`):**
```elixir
def home(conn, _params) do
  render(conn, :home)
end
```

- `conn` = like `req` + `res` combined
- `_params` = like `req.params` + `req.body`

---

## 6. `lib/hello/repo.ex` = Your Database Connection (Sequelize/Knex)

This is your Ecto repository - the interface to PostgreSQL:
```elixir
Hello.Repo.all(User)        # Like User.findAll() in Sequelize
Hello.Repo.get(User, 1)     # Like User.findByPk(1)
Hello.Repo.insert(changeset) # Like user.save()
```

---

## 7. Config Files = Your `.env` + config files

| File | Purpose |
|------|---------|
| `config/config.exs` | Base config (all environments) |
| `config/dev.exs` | Development overrides |
| `config/prod.exs` | Production settings |
| `config/runtime.exs` | Reads env vars at runtime (like `process.env.DATABASE_URL`) |
| `config/test.exs` | Test environment |

In `dev.exs`, the database is configured:
```elixir
database: "hello_dev",
username: "postgres",
password: "postgres",
hostname: "localhost"
```

---

## 8. Templates = Like React Components (sort of)

| Concept | React | Phoenix |
|---------|-------|---------|
| Component files | `.jsx` | `.heex` (HTML + Elixir) |
| Layout wrapper | `<Layout>` component | `layouts.ex` |
| Reusable UI | Component library | `core_components.ex` |
| Props | `props` | `assigns` |

The `.heex` files are like JSX but for Elixir:
```heex
<.button navigate={~p"/users"}>Go to Users</.button>
```

---

## 9. The Folder Structure Mental Model

```
lib/
├── hello/              # BACKEND (Business logic, like your "services" or "models")
│   ├── application.ex  # Starts everything
│   └── repo.ex         # Database connection
│
└── hello_web/          # FRONTEND/WEB LAYER (like your Express routes + React)
    ├── router.ex       # Routes
    ├── endpoint.ex     # Middleware
    ├── controllers/    # Request handlers
    └── components/     # UI components (layouts, buttons, etc.)
```

## Quick Command Cheatsheet

| What you want | Command |
|---------------|---------|
| Install dependencies | `mix deps.get` |
| Create database | `mix ecto.create` |
| Run migrations | `mix ecto.migrate` |
| Start server | `mix phx.server` |
| Interactive console | `iex -S mix` |
| Run tests | `mix test` |
| Generate migration | `mix ecto.gen.migration create_users` |

The server runs on **http://localhost:4000** (not 3000 like Express).
